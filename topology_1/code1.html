1관련 라이브러리 설치

from mininet.net import Mininet
from mininet.cli import CLI
from mininet.node import Host
from mininet.node import OVSKernelSwitch
from mininet.log import setLogLevel, info
from mininet.node import RemoteController
from mininet.term import makeTerm

각 라이브러리의 역할:

  • Mininet: 전체 네트워크 토폴로지를 생성하고 관리하는 메인 클래스
  • CLI: 사용자가 네트워크와 상호작용할 수 있는 대화형 환경 제공
  • Host: 네트워크 상의 엔드포인트(클라이언트, 서버) 역할
  • OVSKernelSwitch: OpenFlow 프로토콜을 지원하는 가상 스위치
  • setLogLevel, info: 로그 출력 레벨 설정 및 정보 메시지 출력
  • RemoteController: 외부 SDN 컨트롤러와 연결하여 중앙집중식 네트워크 제어
  • makeTerm: 각 노드별로 독립적인 터미널 창을 생성

2Mininet 클래스와 net 객체

def myTopo():
  net = Mininet(topo=None, autoSetMacs=False, build=False, ipBase='10.0.1.0/24')

Mininet에서 사용될 Topology를 정의하는 함수

주요 매개변수:

  • topo=None: 사전 정의된 토폴로지를 사용하지 않음 (수동 구성)
  • autoSetMacs=False: MAC 주소 자동 할당 비활성화
  • build=False: 네트워크 즉시 구축하지 않음
  • ipBase='10.0.1.0/24': 호스트들의 기본 IP 주소 대역 설정

3호스트 및 스위치 추가

호스트 추가

h1 = net.addHost('client', cls=Host, defaultRoute=None)
h2 = net.addHost('server1', cls=Host, defaultRoute=None)
h3 = net.addHost('server2', cls=Host, defaultRoute=None)

컨트롤러 및 스위치 추가

c1 = net.addController('c1', RemoteController)
s1 = net.addSwitch('s1', protocols='OpenFlow13', failMode='standalone')
RemoteController는 기본적으로 localhost:6633 포트로 외부 SDN 컨트롤러에 연결

함수 구조:

  • addHost(name, cls, defaultRoute): 호스트 추가
  • addSwitch(name, protocols, failMode): 스위치 추가
  • failMode 옵션: 'standalone' 또는 'secure'

5IP 주소 설정

h1.setIP(intf='client-eth0', ip='10.0.1.5/24')
h2.setIP(intf='server1-eth0', ip='10.0.1.2/24')
h3.setIP(intf='server2-eth0', ip='10.0.1.3/24')

IP 주소 할당:

  • Client (h1): 10.0.1.5/24
  • Server1 (h2): 10.0.1.2/24
  • Server2 (h3): 10.0.1.3/24
  • 서브넷 마스크: 255.255.255.0

6네트워크 제어

네트워크 구축 및 시작

net.build()
net.start()

터미널 생성

net.terms += makeTerm(c1)
net.terms += makeTerm(h1)
net.terms += makeTerm(h2)
net.terms += makeTerm(h3)

CLI 실행 및 종료

CLI(net)
net.stop()
CLI가 실행되면 다음과 같은 테스트 명령어를 사용할 수 있음
mininet> h2 ping h3 (h2에서 h3로 ping 전송)

주요 함수 설명:

  • net.build(): 가상 인터페이스 생성 및 초기화
  • net.start(): 모든 노드와 링크 활성화
  • makeTerm(): GUI 환경에서 별도 터미널 창 생성
  • CLI(): 대화형 네트워크 제어 인터페이스
  • net.stop(): 네트워크 정리 및 종료

7메인 함수

if __name__ == '__main__':
  setLogLevel('info')
  myTopo()

로그 레벨 옵션:

  • debug: 가장 상세한 로그 레벨, 디버깅용
  • info: 일반적인 정보를 제공하는 로그 레벨
  • output: 출력 결과를 로그로 기록
  • warning: 경고 메시지를 로그로 기록
  • error: 오류 메시지를 로그로 기록
실행 결과: 리눅스 기반 운영체제에서 실행 시 Mininet CLI가 활성화되며 네트워크 명령을 통해 네트워크 제어 및 테스트 가능

8주요 구조 및 의존성

주요 구조

  • Library Import: Mininet 핵심 모듈들
  • Network 생성: 호스트 3개 + 스위치 1개 + 컨트롤러 1개
  • Topology: 스타 구조 (모든 호스트가 중앙 스위치에 연결)
  • 설정: 수동 MAC/IP 주소 할당
  • 제어: CLI를 통한 대화형 네트워크 제어

주요 의존성

  • Open vSwitch: 가상 스위치 구현
  • Python: 스크립팅 환경
  • Linux Network Namespace: 네트워크 격리
  • iptables: 방화벽 및 NAT 기능
